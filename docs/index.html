Módszertan és adatforrások

A becslés óránkénti időjárási és naptári adatokon alapul. Minden nyitvatartási órára négy független szorzótag szorzataként számolunk kapacitásarányt (0–1), mellyel az alap óradíjat megszorozzuk:

Bevétel = Alap óradíj × Mnaptár × Mhőmérséklet × Mcsapadék × Mfront
1. Mnaptár — Nyitvatartási és naptári szorzó
  • Hétvégén (szombat–vasárnap) és magyar ünnepnapokon: nyitva 10:00–18:00, szorzó = 1,0
  • Munkanapokon (hétfő–péntek): nyitva 12:00–20:00
    • 12:00–15:59 → szorzó = 0,2 (alacsony déli forgalom)
    • 16:00–19:59 → szorzó = 0,6 (délutáni csúcsforgalom)
  • Nyitvatartáson kívüli órákon: szorzó = 0
2. Mhőmérséklet — Hőmérsékleti szorzó
  • 21–25 °C: 1,0 — optimális szabadtéri közérzet
  • 16–21 °C: lineárisan nő 0-ról 1-re — hűvös, de vállalható
  • 25–30 °C: lineárisan csökken 1-ről 0-ra — hőség-határ
  • 16 °C alatt vagy 30 °C felett: 0 — nincs számított bevétel
3. Mcsapadék — Csapadék szorzó
  • 0 mm/óra (száraz): 1,0
  • 0–1 mm/óra (szitálás, permet): 0,5
  • >1 mm/óra (eső): 0,1 — drasztikus forgalomcsökkenés
4. Mfront — Légnyomásfront szorzó
  • Ha az előző 3 óra légnyomásváltozása |ΔP| ≥ 3 hPa: szorzó = 0,75
    Front átvonulása fejfájást, fáradtságot okoz — mérhető vendégforgalom-csökkenés.
  • Egyéb esetben: szorzó = 1,0

Adatforrások (ellenőrizhető):

  • Órás időjárási archív (hőmérséklet, csapadék, légnyomás) — Open-Meteo Historical Weather API · Élő API példa (2024. június)
  • Magyar ünnepnapok — python-holidays könyvtár (forrás: Magyar Közlöny)
